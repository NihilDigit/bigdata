# Weather Lab 气象大数据计算与可视化系统设计文档

## 1. 项目范围

Weather Lab 是《大数据计算与可视化》课程设计项目。系统覆盖气象数据采集、传输、存储、处理和可视化，保留课程要求中的 HBase、HDFS、Hive、Spark 链路，Web 层改用 FastAPI 与 Next.js。

系统包含两个页面：

- `总览页`：中国轮廓地图、站点天气点位、当前站点五指标读数。
- `详细数据页`：趋势图、统计摘要、数据记录表。

系统固定三个站点：

| 站点 | 作用 | 实时数据来源 | 历史数据来源 |
|---|---|---|---|
| 唐山 | 硬件演示站 | ESP32 温湿度；Open-Meteo 气压、风速、风向、天气代码 | Open-Meteo |
| 北京 | 对比站 | Open-Meteo | Open-Meteo |
| 上海 | 对比站 | Open-Meteo | Open-Meteo |

核心技术选型：

| 层级 | 技术 |
|---|---|
| 硬件采集 | ESP32-C3、DHT11 |
| 采集服务 | Python、TCP Socket |
| 实时存储 | HBase |
| 历史存储 | HDFS |
| 数据仓库 | Hive 外部表 |
| 批处理分析 | Spark on YARN、PySpark |
| Web API | FastAPI |
| 前端 | Next.js、shadcn/ui、Recharts |
| 地图 | 官方合规中国轮廓图 |

需要在课程报告中说明的替换项：

- FastAPI 替换 MTAS。
- Next.js + shadcn/ui + Recharts 替换静态 HTML + ECharts。
- Open-Meteo 真实气象数据替换随机仿真气压、风速、风向。

## 2. 总体架构

```text
ESP32-C3 + DHT11
    |
    | TCP CSV
    v
Python 采集服务
    |
    |---------------------> HBase realtime_weather
    |                              |
    |                              v
    |                         FastAPI 当前数据
    |
    |---------------------> HDFS /weathertextdb
                                   |
                                   v
                              Hive weather_table
                                   |
                                   v
                              Spark 分析任务
                                   |
                                   v
                         HDFS /weather_analysis/*
                                   |
                                   v
                            FastAPI 分析数据
                                   |
                                   v
                         Next.js 可视化界面
```

实时链路：

```text
ESP32 温湿度
+ Open-Meteo 当前气象
-> Python 采集服务
-> HBase
-> FastAPI
-> 总览页
```

历史分析链路：

```text
Open-Meteo 历史数据
-> HDFS
-> Hive
-> Spark
-> HDFS 分析结果
-> FastAPI
-> 详细数据页
```

## 3. 数据模型

系统内部观测记录：

```csv
collect_time,station_id,station_name,temperature,humidity,pressure,wind_speed,wind_direction,weather_code
```

字段定义：

| 字段 | 类型 | 含义 | 单位 |
|---|---|---|---|
| collect_time | timestamp | 采集时间，Asia/Shanghai | - |
| station_id | string | 站点 ID | tangshan / beijing / shanghai |
| station_name | string | 站点名称 | 唐山 / 北京 / 上海 |
| temperature | double | 温度 | 摄氏度 |
| humidity | double | 相对湿度 | % |
| pressure | double | 地面气压 | hPa |
| wind_speed | double | 10 米风速 | m/s |
| wind_direction | double | 10 米风向 | 度 |
| weather_code | int | 天气代码 | Open-Meteo weather code |

课程兼容 CSV：

```csv
collect_time,station_name,temperature,humidity,pressure,wind_speed,wind_direction
```

示例：

```csv
2026-06-18 10:45:21,唐山,28.5,64,1003.6,2.39,200
```

Open-Meteo 字段映射：

| Open-Meteo 字段 | 系统字段 |
|---|---|
| temperature_2m | temperature |
| relative_humidity_2m | humidity |
| surface_pressure | pressure |
| wind_speed_10m | wind_speed |
| wind_direction_10m | wind_direction |
| weather_code | weather_code |

天气代码展示映射：

| weather_code | 展示 |
|---|---|
| 0 | 晴 |
| 1, 2, 3 | 少云 / 多云 |
| 45, 48 | 雾 |
| 51, 53, 55 | 毛毛雨 |
| 61, 63, 65 | 雨 |
| 71, 73, 75, 77 | 雪 |
| 80, 81, 82 | 阵雨 |
| 95, 96, 99 | 雷暴 |

## 4. 数据采集

### 4.1 ESP32 采集

硬件：

- ESP32-C3 开发板。
- DHT11 温湿度传感器。
- USB 数据线。
- 杜邦线。

DHT11 接线：

| DHT11 | ESP32-C3 |
|---|---|
| VCC | 3.3V |
| GND | GND |
| DATA | GPIO4 |

ESP32 只采集温度、湿度。气压、风速、风向不在硬件端生成。

ESP32 发送格式：

```csv
tangshan,唐山,28,64,1
```

最后一列为 ESP32 侧递增样本号，用于前端展示实时更新；DHT11 温湿度为整数级读数。

采样周期：

```text
1 秒
```

### 4.2 Open-Meteo 当前数据

服务器侧请求 Open-Meteo 当前天气接口。采集服务缓存最近一次结果。

刷新周期：

```text
5 到 15 分钟
```

唐山站实时合并规则：

| 指标 | 来源 |
|---|---|
| 温度 | ESP32 + DHT11 |
| 湿度 | ESP32 + DHT11 |
| 气压 | Open-Meteo |
| 风速 | Open-Meteo |
| 风向 | Open-Meteo |
| 天气代码 | Open-Meteo |

北京、上海实时数据全部来自 Open-Meteo。

### 4.3 历史数据导入

开发截止日前导入三个站点的历史数据。

导入步骤：

1. 请求 Open-Meteo Historical Weather API。
2. 规范化为课程兼容 CSV。
3. 上传到 HDFS `/weathertextdb`。
4. 创建 Hive 外部表。
5. 运行 Spark 分析任务。

历史数据用于详细数据页的趋势、统计摘要和记录表。

## 5. HBase 设计

HBase 保存短期实时记录。

表名：

```text
realtime_weather
```

列族：

```text
data
```

行键：

```text
station_id_timestamp
```

示例：

```text
tangshan_20260618104521
```

列：

```text
data:station_name
data:temperature
data:humidity
data:pressure
data:wind_speed
data:wind_direction
data:weather_code
data:collect_time
```

TTL：

```text
150 秒
```

按课程指导文档示例设置为 150 秒，报告中说明 HBase 保存短期实时数据。

## 6. HDFS 与 Hive 设计

HDFS 原始数据路径：

```text
/weathertextdb
```

Spark 分析结果路径：

```text
/weather_analysis/window_mean
/weather_analysis/summary
```

课程指导文档路径兼容：

```text
/weather_10secmean
```

Hive 外部表：

```sql
CREATE EXTERNAL TABLE IF NOT EXISTS weather_table (
    collect_time TIMESTAMP COMMENT '采集时间',
    station_name STRING COMMENT '气象站名称',
    temperature DOUBLE COMMENT '温度，摄氏度',
    humidity DOUBLE COMMENT '相对湿度，百分比',
    pressure DOUBLE COMMENT '地面气压，hPa',
    wind_speed DOUBLE COMMENT '10 米风速，m/s',
    wind_direction DOUBLE COMMENT '10 米风向，角度'
)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
LOCATION '/weathertextdb';
```

验证 SQL：

```sql
SELECT * FROM weather_table LIMIT 10;
```

## 7. Spark 分析设计

Spark 从 Hive 表读取 HDFS 原始数据，输出可视化所需的聚合数据。

### 7.1 窗口均值

输入：

```text
秒级或分钟级原始观测记录
```

处理：

```text
按 station_name 和时间窗口分组
计算 temperature、humidity、pressure、wind_speed、wind_direction 的均值
```

输出：

```csv
window_start,station_name,temperature,humidity,pressure,wind_speed,wind_direction
```

默认窗口：

```text
10 秒
```

可选粒度：

```text
10 秒
1 小时
1 天
```

### 7.2 统计摘要

输出字段：

```text
平均温度
最高温度
最低温度
平均湿度
平均气压
气压变化量
平均风速
最大风速
主导风向
```

### 7.3 多站点对比

输入：

```text
同一指标、同一时间范围、三个站点
```

输出：

```text
唐山序列
北京序列
上海序列
```

前端以三条线展示。

### 7.4 风向处理

风向是角度数据。普通平均在跨 0 度时会产生错误。

推荐算法：

```text
u = mean(cos(theta))
v = mean(sin(theta))
direction = atan2(v, u)
```

最低版本可以使用普通平均，并在报告中说明局限。

### 7.5 防重复写入

Spark 定时重算时避免重复 append。

推荐路径：

```text
/weather_analysis/window_mean/date=2026-06-18/hour=10
```

同一分区重算时覆盖旧结果。

## 8. FastAPI 接口

### 8.1 当前站点数据

```text
GET /api/stations/current
```

用途：

```text
总览页地图点位
总览页五指标读数框
```

响应示例：

```json
[
  {
    "id": "tangshan",
    "name": "唐山",
    "latitude": 39.63,
    "longitude": 118.18,
    "time": "2026-06-18T10:45:21+08:00",
    "temperature": 28.5,
    "humidity": 64,
    "pressure": 1003.6,
    "wind_speed": 2.39,
    "wind_direction": 200,
    "weather_code": 3
  }
]
```

### 8.2 最近实时序列

```text
GET /api/stations/{station_id}/live?seconds=150
```

用途：

```text
详细数据页最近记录和短期趋势
```

### 8.3 分析趋势

```text
GET /api/analysis/trends
```

查询参数：

| 参数 | 值 |
|---|---|
| station_id | tangshan / beijing / shanghai / all |
| metric | temperature / humidity / pressure / wind_speed / wind_direction |
| range | 24h / 7d / custom |
| granularity | 10s / 1h / 1d |

### 8.4 原始记录

```text
GET /api/records
```

用途：

```text
详细数据页数据记录表
```

## 9. 前端设计

前端按天气信息组织，不按后端组件组织。页面不显示 HBase、HDFS、Hive、Spark 的在线状态。

### 9.1 总览页

页面结构：

```text
Weather Lab / 气象数据工作台


+--------------------------------------------------------------------------------+
|                                                                    [刷新]       |
|                                                                                |
|                              中国轮廓地图                                       |
|                                                                                |
|                    晴 北京                                                     |
|                      31.2 C                                                     |
|                                                                                |
|                         多云 唐山                                               |
|                              28.5 C  selected                                  |
|                                                                                |

|                                           27.9 C                               |
|                                                                                |

+--------------------------------------------------------------------------------+
| 唐山                                                          10:45:21          |
```

### 9.3 视觉规范

基础风格：

- 浅色工作台界面。
- 冷灰白背景。
- 石墨色正文。
- 低对比细边界。
- 8px 以内圆角。
- 少阴影。

指标颜色：

| 指标 | 颜色方向 |
|---|---|
| 温度 | 低饱和橙红 |
| 湿度 | 低饱和蓝 |
| 气压 | 紫灰 |
| 风速 | 绿色 |
| 风向 | 靛蓝 |

避免：

- 大屏科技蓝。
- 高饱和渐变。
- 玻璃拟态。
- 厚重阴影。
- 后台管理模板侧栏。

### 9.4 响应式

| 设备 | 布局 |
|---|---|
| 桌面 | 地图撑满主内容区；五指标一行；详细页图表两列 |
| 平板 | 地图高度降低；五指标 3 + 2；图表可降为单列 |
| 手机 | 地图标签缩短；五指标两列或单列；图表单列 |

## 11. 运行流程

历史数据准备：

1. 拉取唐山、北京、上海 Open-Meteo 历史数据。
2. 转换为课程兼容 CSV。
3. 上传 HDFS `/weathertextdb`。
4. 创建 Hive 外部表。
5. 运行 Spark 分析任务。

服务启动：

1. 启动 HDFS。
2. 启动 YARN。
3. 启动 HBase。
4. 启动 HBase Thrift。
5. 启动 Hive。
6. 启动采集服务。
7. ESP32 上电并连接 Wi-Fi。
8. 启动 FastAPI。
9. 启动 Next.js。

## 12. 验收标准

数据链路：

- ESP32 能发送唐山温湿度。
- 采集服务能合并 Open-Meteo 当前气象数据。
- HBase 能查到实时记录。
- HDFS 能查到历史记录和实时追加记录。
- Hive 能查询 `weather_table`。
- Spark 能生成窗口均值和摘要结果。

前端：

- 总览页显示合规中国轮廓图。
- 北京、唐山、上海三个站点可点击。
- 地图点位显示天气状态和温度。
- 五指标框随站点切换。
- 详细数据页显示趋势、统计摘要和记录。
- 移动端内容不重叠。
