## 2. 采集方案：三站统一 WebSocket + 扰动

**日期**：2026-06-25

**决定**：三个站点使用统一抽象，以 WebSocket Client 方式连接采集服务器，1s 推送频率。

### 目标数据流

```
ESP32（唐山）WS Client  ──┐
                          │
北京模拟 WS Client       ──┼──→ 采集服务器 WS Server :8080
（Open-Meteo + 扰动）     │         │
                          │         ├──→ HBase（实时，TTL 150s）
上海模拟 WS Client       ──┘         │
（Open-Meteo + 扰动）               └──→ HDFS /weathertextdb（周期落盘）
```

### 三个站的定义

| 站点 | 实现方式 | 数据源 | 频率 |
|---|---|---|---|
| 唐山 | ESP32-C3 MicroPython WS Client | DHT11 温湿度 + Open-Meteo 补气压/风速/风向/天气码 | 1s |
| 北京 | 主机 Python WS Client | Open-Meteo 当前值 + 小幅度扰动 | 1s（Open-Meteo 基准值每 5-15min 刷新） |
| 上海 | 主机 Python WS Client | Open-Meteo 当前值 + 小幅度扰动 | 同上 |

### 扰动规则

| 字段 | 扰动幅度 |
|---|---|
| 温度 | ±0.3°C |
| 湿度 | ±2% |
| 气压 | ±0.5 hPa |
| 风速 | ±0.5 m/s |
| 风向 | ±5° |

报告说明：北京/上海使用 Open-Meteo 真实数据作为基准，加小幅度模拟扰动使曲线连续。

### 需要新增/修改

- 新增：`采集服务器 WS Server`（替换 `weather_dataserver.py` 的 TCP Client 模式）
- 新增：`openmeteo_station_client.py`（北京/上海模拟站，WS Client + Open-Meteo + 扰动）
- 修改：`esp32/weatherstation_main.py`（从 TCP Server 改为 WS Client）
- 修改：`weather_dataserver.py` → 改为 WS Server，同时接收三个站的推送
- 各站发送的数据格式统一为同一种 CSV 或 JSON 协议

### 不在此范围内的

- 前端推送方式不变（仍为 polling 或后续再加 WebSocket）
- Open-Meteo 历史数据获取方式不变

## 5. Spark 分析：YARN + 手动刷新 + 缓存保留

**日期**：2026-06-25

**决定**：Spark 部分保持现有实现不变，触发方式为手动刷新。

### 当前链路（保持不变）

```
HDFS /weathertextdb
  → Hive 外部表 weather_table
  → Spark on YARN（手动 spark-submit）
  → HDFS /weather_analysis/*
  → collect 到 data/processed/*.json
  → FastAPI 读 JSON → 前端
```

### 触发方式

- **演示前**：手动跑 `run-spark-analysis-yarn.sh` 一次，算出历史趋势和统计摘要
- **演示中**：前端手动刷新 → 异步触发新的 `spark-submit` → 旧数据继续展示 → Spark 完成后自动替换
- **缓存**：Spark 结果写入 HDFS 和本地 JSON，前端始终读到最新完成的结果

### 不需要修改

- `spark_weather_analysis.py` 不变
- `run-spark-analysis-yarn.sh` 不变
- YARN 启动脚本不变

### 报告中需注明

- Spark 通过 `--master yarn` 提交到 YARN 集群运行
- 可通过前端手动触发重算（加分项）
- 结果经 HDFS 持久化后 collect 到本地 JSON 供 FastAPI 读取

## 6. Spark 分析内容补充

**日期**：2026-06-25

**决定**：补充分析内容，对齐设计文档 §7。

### 当前缺口

| 分析项 | 设计文档写了 | 实际实现了 |
|---|---|---|
| 统计摘要字段 | 平均/最高/最低温度、均湿、**气压变化量**、**最大风速**、**主导风向** | ✅ 均温/最高/最低/均湿/均压/均风速 ❌ 气压变化量、最大风速、主导风向 |
| 分区防重复 | `/weather_analysis/window_mean/date=.../hour=...` | ❌ 使用 `mode("overwrite")` 整体覆盖 |
| 风向处理 | 推荐 u/v 分解 | ❌ 使用普通平均 |

### 决定

- **统计摘要**：补充 气压变化量、最大风速、主导风向 三个字段
- **分区防重复**：保持现状（`mode("overwrite")`），不动分区结构
- **风向处理**：保持现状（普通平均），报告中说明局限
